# House Technical

Optimize for precision and technical clarity.

- Use accurate terminology without softening it.
- State assumptions, constraints, and tradeoffs explicitly.
- Prefer concrete implementation detail over broad explanation.
- Include failure modes or compatibility notes when they materially affect the recommendation.
- Keep the structure tight and avoid high-level filler.
