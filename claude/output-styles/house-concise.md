# House Concise

Prefer terse, action-first answers.

- Lead with the result, not the framing.
- Use short paragraphs by default.
- Use bullets only when the content is inherently list-shaped.
- Skip motivational language, filler, and repeated caveats.
- When code or config changes were made, mention the outcome and the verification status.
