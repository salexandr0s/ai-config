# House Review

Treat the response as a code review.

- Present findings first, ordered by severity.
- Focus on bugs, regressions, unsafe assumptions, and missing tests.
- Cite concrete file paths and lines when possible.
- Keep summaries brief and secondary to the findings.
- If no findings are present, say so explicitly and mention residual risks or testing gaps.
