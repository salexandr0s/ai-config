## Mode Detection

Detect from $ARGUMENTS (or ask if unclear):

- **Expansion** — Dream big. What's the 10-star version? What adjacent opportunities exist?
- **Selective Expansion** — Hold current scope but cherry-pick 1-2 high-value additions worth the marginal cost
- **Hold Scope** — Maximum rigor on the current plan. No additions. Focus on: is this the right thing to build?
- **Reduction** — Strip to absolute essentials. What can be cut without losing the core value?

Default to **Hold Scope** if no mode specified.

---

## Review Process

For the current plan (read from context, task list, or ask for it):

### 1. Premise Challenge
- What assumptions does this plan make about users, market, or technology?
- Which assumptions are validated vs hoped-for?
- What happens if the key assumption is wrong?

### 2. Opportunity Cost
- What are we NOT building by doing this?
- Is there something higher-leverage we could do instead?
- What's the cost of waiting 3 months to build this?

### 3. Mode-Specific Analysis

**Expansion:**
- What would make this 10x more valuable?
- What adjacent problems could this solve?
- What partnerships or integrations multiply the value?

**Selective Expansion:**
- List 3-5 potential additions ranked by value/effort
- For each: marginal effort, marginal value, risk of scope creep
- Recommend 0-2 additions worth including

**Hold Scope:**
- Is every feature in the plan necessary for the core value?
- Are there simpler ways to achieve the same outcome?
- What's the minimum that ships and validates the hypothesis?

**Reduction:**
- For each feature: what breaks if we remove it?
- What's the absolute minimum viable version?
- What can be deferred to v2 without losing v1 value?

### 4. Verdict

Format:
